Dev Web Trends 2018

TREND #1: Mengerti Dasar

Pengetahuan dasar mengenai HTML, CSS, dan JavaScript akan tetap menjadi hal terpenting yang harus Anda ketahui sebagai pengembang web pada tahun 2018!
dengan dasar-dasar ini kita memiliki modal untuk mempelajari lebih banyak teknologi untuk pengembangan web.
Kita tidak perlu menguasai semua teknologi pengembangan web tapi kita perlu mengetahui bagaimana HTML, CSS, dan JavaScript bekerja dan bagaimana mereka bekerja sama.

TREND #2: Node.js

Node.js adalah bahasa sisi server yang harus Anda lihat. Dan Anda tahu dasar-dasarnya (JavaScript!) bukan? Node.js hanya JavaScript di server jadi mengapa tidak menyelam lebih dalam ke dalamnya dan menjadi pengembang frontend dan backend?
Selain itu, tidak ada satu keuntungan besar yang bisa membuat Node.js benar-benar lebih baik daripada PHP dll. Ini adalah bahasa yang menawarkan kinerja hebat. Ini menampilkan ekosistem yang dinamis dan aktif dengan banyak paket pihak ketiga yang dapat Anda pasang ke aplikasi Anda. Tapi alasan #1 mengapa sangat populer adalah JavaScript menjadi lebih dan lebih penting.

TREND #3: Eksplorasi React, Angular dan / atau Vue.js (Frontend Frameworks)

React, Angular dan Vue.Js adalah framework yang digunakan untuk client-side atau yang juga biasa disebut front-end dengan menggunakan basis bahasa javasript, tetapi react tidak bisa dikatakan sebagai framework, lebih tepatnya adalah library.Ketiga hal di atas menciptakan pengalaman pengguna/user experience yang sangat menarik untuk aplikasi web kamu karena semuanya dikontrol dengan JavaScript. Pengguna mungkin terus menyukai aplikasi web yang menarik, sehingga kerangka kerja frontend seperti Angular, React atau Vue.js akan tetap penting di tahun 2018. Mempelajari salah satu atau semuanya akan menjadi ide yang sangat bagus. Mereka memungkinkan dirimu untuk belajar tentang web developmnet dan menciptakan web yang bagus dengan interface yang interaktif

TREND #4: Web Components & Component Compilers

Ide pada web components itu sederhana yaitu untuk membangun komponen yang hanya berfungsi di library itu sendiri, sedangkan dengan menggunakan vanilla JavaScript dapat membangun komponene web yang juga dapat digunakan luar framework.

TREND #5: Use Static Page Generators

Trend berikutnya di tahun 2018 adalah Penggunaaan Static Page Generator, Static Page Generator adalah aplikasi statis yang memungkinkan kita untuk membuat halaman web pada server lokal atau pun built-in server yang berisikan Html,Css dan Javasript. Keuntungan dari Static Page generator adalah, karena sudah multi platform dan juga dapat menggunakan markdown. Contoh beberapa Generator Static Page adalah Jekyll, Hugo, Nuxt.js dan lain-lain

TREND #6: Progressive Web Apps (PWAs)

Progressive Web Apps (PWA) merupakan istilah yang mencakup beberapa teknologi yang memungkinkan untuk membuat aplikasi web yang berperilaku sangat mirip dengan aplikasi mobile.

Keuntungannya adalah karena mencakup fungsi offline (misalnya, bagian dari aplikasi web yang dapat berfungsi tanpa koneksi Asalkan telah dimuat satu kali) penggunaan perangkat asli fitur (seperti mengakses kamera perangkat).

Kekurangannya adalah kamu harus mempelajari dua bahasa (Java / Android dan Swift / ObjectiveC) dan jangkauannya punan terbatas.

TREND #7: Build Serverless Apps

Serverless bukan berarti tidak menggunakan server sama sekali, namun menggunakan server hanya jika diperlukan saja, sehingga tidak perlu khawatir mengelola dan memperbarui server. Space pada server pun akan menjadi lebih banyak. Biaya yang dikeluarkan pun lebih sedikit karena tidak perlu membayar saat dalam keaadan idle.

TREND #8: Artificial Intelligence and Bots

AI adalah bidang yang luas Kamu bisa menyelam ke Python, Machine Learning, Big Data, Deep Learning dan membangun alat AI sendiri. Jika Kamu menguasai semua itu, Kamu akan memiliki masa emas di depan Kamu.
Namun, untuk penggunaan lainnya, kami dapat menyelami lebih banyak alat yang memungkinkan kami menerapkan layanan AI ke semua jenis aplikasi dan model bisnis tanpa persyaratan menjadi ahli AI.
Pertimbangkan alat seperti AWS Lex, layanan yang memungkinkan Kamu membangun bots obrolan cerdas yang dapat Kamu terapkan ke halaman Kamu. Alat seperti ini membuat AI dapat diakses oleh kita semua dan memungkinkan Kamu untuk berfokus pada ide dan logika bisnis Kamu. Dan kita akan melihat banyak alat seperti ini muncul (dan menjadi lebih baik) di tahun 2018 - oleh karena itu ini adalah tren yang mungkin tidak ingin Kamu lewatkan.

Git - Simple Guide

Ini adalah pedoman simple untuk mulai menggunakan git

Tweet this!

original by Roger Dudler
translate by Azhar Khaira

Setup

Download git untuk OSX
Download git untuk Windows
Download git untuk Linux

Membuat repositori baru

buat direktori baru, buka, dan lakukan
git init
untuk membuat repository git baru.

Memeriksa repositori

buat salinan pekerjaan dari local repository dengan menjalankan perintah
git clone /path/to/repository
ketika menggunakan server jarak jauh, isi perintahnya akan menjadi
git clone username@host:/path/to/repository

Alur kerja

Repositori lokal kamu terdiri dari tiga bagian dengan sebutan “trees“
yang dikelola oleh git. Yang pertama adalah *Working Directory*
yang menyimpan berkas aktual. Kedua adalah *Index* yang
berperan sebagai pengolah data dan yang ketiga *Head*
yang mengarah pada komit terakhir.

Tambah dan komit

Kamu bisa melakukan perubahan (penambahan ke index) menggunakan
git add <filename>
git add *
ini merupakan langkah awal alur kerja dasar git.
Untuk komit sepenuhnya gunakan
git commit -m "commit message"
Sekarang berkas sudah berkomit di head,
tapi belum di repositori jarak jauh.

Mengirim perubahan

Saat ini perubahan sudah tersimpan di head salinan kerja lokal kamu.
Untuk mengirimkan ke repositori jarak jauh, lakukan
git push origin master
Ubah master sesuai dengan cabang yang kamu inginkan.

Jika repositori yang ada belum dikloning dan ingin dihubungkan
ke server jarak jauh, kamu perlu menambahkan
git remote add origin <server>
Sekarang kamu bisa mengirimkan perubahan ke server jarak jauh yang dituju.

Percabangan

Percabangan atau branching digunakan untuk mengembangkan
fitur-fitur secara terisolasi. Cabang utama atau master merupakan
cabang bawaan ketika kamu membuat repositori. Gunakan cabang lain
untuk pengembangan, setelah selesai, gabungkan kembali ke cabang utama.

buat cabang baru dengan nama “fitur_x” dan beralih kedalamnya menggunakan
git checkout -b fitur_x
beralih kembali ke master
git checkout master
dan hapus cabang yang tadi dibuat
git branch -d fitur_x
suatu cabang tidak terbuka untuk yang lainnya kecuali
jika kamu mengirimkannya ke repositori jarak-jauh.
git push origin <cabang>

Perbaruan dan penggabungan

Untuk memperbarui repositori lokal ke komit terkini, lakukan
git pull
dari direktori kerja kamu untuk
mengambil dan menggabungkan perubahan jarak-jauh.
Untuk menggabungkan cabang lain ke cabang aktif (misal master), gunakan
git merge <cabang>
pada kasus diatas, git mencoba menggabungkan perubahan secara
otomatis. Sayangnya hal ini tak selalu berjalan mulus dan bisa
menyebabkan konflik. Kamu lah yang bertanggung jawab
menggabungkan konflik tersebut secara manual dengan menyunting
berkas yang ditunjukkan git. Setelah itu, kamu perlu menandainya dengan
git add <filename>
sebelum penggabungan berlaku, kamu bisa melakukan pratinjau menggunakan
git diff <cabang_asal> <cabang_tujuan>

Penandaan

Sangat dianjurkan membuat penanda atau tags untuk perangkat lunak
yang dirilis. Hal ini amat lah lazim, yang juga terjadi di SVN. Kamu bisa
membuat penanda baru dengan nama 1.0.0 dengan menjalankan
git tag 1.0.0 1b2e1d63ff
1b2e1d63ff adalah 10 karakter pertama dari identitas komit yang ingin
kamu referensikan ke penanda. Kamu bisa mendapatkan identitas
komit dengan melihat LOG

Log

Dalam bentuknya yang paling sederhana, kamu bisa mempelajari
riwayat repositori menggunakan
git log
kamu bisa menambahkan banyak parameter untuk menampilkan log
sesuai keinginan. Untuk melihat komit penulis tertentu:
git log --author=bob
Untuk melihat log yang dimampatkan, satu baris per komit:
git log --pretty=oneline
Atau mungkin kamu ingin melihat pohon ASCII art seluruh
percabangan disertai nama dan penandanya:
git log --graph --oneline --decorate --all
Sekedar melihat berkas yang berubah:
git log --name-status
Ini baru sedikit saja dari sekian banyak parameter yang bisa kamu
gunakan. Lebih jauh lagi, lihat
git log --help

Mengembalikan perubahan lokal

Seandainya kamu melakukan kesalahan, kamu bisa
mengembalikannya menggunakan perintah
git checkout -- <namaberkas>
perintah di atas mengembalikan perubahan di dalam pokok kerja kamu
dengan konten terakhir dari HEAD. Perubahan dan berkas baru yang
telah ditambahkan ke indeks akan tetap tersimpan.

Jika kamu ingin membatalkan perubahan dan komit lokal seutuhnya,
ambil riwayat terakhir dari server dan arahkan ke cabang master lokal
seperti ini
git fetch origin
git reset --hard origin/master

Petunjuk berguna

GUI git bawaan
gitk
menggunakan output git penuh warna
git config color.ui true
menunjukkan log satu baris per komit
git config format.pretty oneline
menggunakan penambahan interaktif
git add -i
